# dotfiles

## 要件
.zshrc, vscode settings.json, cursor settings.jsonとかをgit管理したい

専用ツールにはあまり依存せず、shell scriptとmakefileぐらいでbareで管理したい

さらに、物によってはファイル分割に対応させたい  
.zshrcをエイリアス設定の部分、パス設定の部分、個々の環境依存の部分に分けるとか。  

また、そういうファイル分割に対応させると、ファイルの編集がめんどくさくなりそうではある。  
たとえば今まで.zshrcを直接開くだけでよかったのが、エイリアス用の設定部分、PATH用の設定部分と開くべきファイルを決めてから開く必要が出てくる。
そこは、makefileなりなんなり何らかの方法で楽に見れるようにしたい。
少なくとも、シンボリックリンクの.zshrc側を直接編集するということはできなくなるだろうから。
あるいは、シンボリックリンクさえ張らずに、毎回 ~/.zshrc を削除して、新たに分割ファイルからマージして作ったものと置き換えるって方法もありそう。どちらが良いだろう？
