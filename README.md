# dotfiles

## このリポジトリで実現したかったこと

- **何を管理するか**  
  .zshrc やエディタ（VSCode / Cursor）の設定などを git で管理し、環境が変わっても同じ設定を再現できるようにしたい。

- **どのように管理するか**  
  専用の dotfiles ツールには頼らず、shell スクリプトと Makefile 程度で、できるだけ「bare」に管理したい。

- **分割と編集のしやすさ**  
  設定によってはファイルを分割したい（例: .zshrc を役割ごと）。分割すると「いまどれを編集すればよいか」が分かりにくくなるので、  
  「何がどこにあるか」を名前・説明付きで一覧でき、編集するときはその名前で指定して開けるようにしたい。  
  必要ならパスだけ取り出して別のエディタで開くこともできるが、その責務はツールではなく利用側に持たせてよい。

- **環境ごとに読み込むものを切り替えたい**  
  マシンや用途（自宅 / 大学 / 会社など）によって、読み込む設定を切り替えたい。  
  「いまどの環境か」は 1 つに決まっており、その環境用の設定だけが読まれる形にしたい。  
  環境の指定は、マシンごとに「いまの環境」を一度セットすればよく、毎回指定しなくてよい形にしたい。  
  また、環境という概念は .zshrc に限らず、将来ほかの設定でも共通して使えるようにしたい。

- **運用の手軽さ**  
  make などの操作は、リポジトリに cd しなくても、どこにいても実行できるようにしたい。
